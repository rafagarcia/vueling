Las tres generaciones de frameworks JS

Web components

Son distintas tecnologías independientes que unidas construyen elementos de interfaz de usuario reusables que son creados usando tecnología Web abierta (Reusables).

Son parte del navegador, y por lo tanto no necesitan bibliotecas externas (Nativos).

Un componente web puede ser usado sin escribir código, simplemente añadiendo una sentencia para importarlo en una página HTML (Declarativos).

Web Components usa capacidades estándar, nuevas o aún en desarrollo, del navegador (Estándar).

Reusable: reutilizables tantas veces como sea necesario

Small: pequeños para aprovechar al máximo su capacidad

Testable: testeables para minimizar errores en su uso a lo largo del ciclo de vida de la aplicación en la que se integren

Reactividad (Data-binding)


Mapeo de las propiedades de la instancia de un objeto de datos con su localización en una fuente de datos.

<!-- HTML -->
<div id="app">
  <p>{{ message }}</p>
  <input v-model="message">
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    message: 'Hello Vue!'
  }
})
</script>

MVVM (Model-View-viewmodel)


Sucesor del patrón MVC.

Provee la capacidad de usar bindings para transferir datos desde y hacia la vista, por lo que el controlador solo es responsable de implementar el comportamiento de la vista. En este caso, el controlador es llamado modelo-vista (view model), dando origen al patrón MVVM

La ventaja frente al MVC viene al usar binding declarativo para transferir los datos desde y hacia el modelo a la vista.

Ciclo de vida de una instancia Vue


Una instancia Vue evoluciona, recorriendo una serie de pasos a lo largo de su instanciación.

Por ejemplo, al ser creada, necesita establecer la observación de su $data, montar la instancia en el DOM y modificarlo cuando $data cambia.

En este camino, ejecuta funciones llamadas "lifecycle hooks" (Hooks del ciclo de vida), permitiendo ejecutar piezas de código en cada uno de esos estados de la instancia.

Hay diferentes hooks a los que engancharse para ejecutar nuestro código: created, mounted, updated o destroyed.

<!-- JS -->
new Vue({
  data: {
    a: 1
  },
  created: function () {
    // `this` points to the vm instance
    console.log('a is: ' + this.a)
  }
})
// => "a is: 1"

Directivas


Las Directivas vienen prefijadas con v- para indicar que son atributos especiales que aplican un comportamiento reactivo al DOM renderizado.

<!-- HTML -->
<div id="app-2">
  <span v-bind:title="message">
    Hover your mouse over me for a few seconds
    to see my dynamically bound title!
  </span>
</div>

<!-- JS -->
var app2 = new Vue({
  el: '#app-2',
  data: {
    message: 'You loaded this page on ' + new Date().toLocaleString()
  }
})

Directivas: v-model


v-model enlaza el elemento del DOM al que se asigna la directiva con el atributo seleccionado de la propiedad data dentro de nuestra instancia vue

Se puede aplicar a los elementos input, select, textarea, o a un componente.

<!-- HTML -->
<div id="app">
  <select v-model="selected">
    <option disabled value="">Please select one</option>
    <option>A</option>
    <option>B</option>
    <option>C</option>
  </select>
  <span>Selected: {{ selected }}</span>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    selected: ''
  }
})
</script>

Directivas: v-show


v-show oculta o muestra el elemento DOM al que se le asigna en función del valor booleano del modelo que seleccionemos

A diferencia de v-if, siempre se renderiza y permanece en el DOM, modificándose la propiedad CSS display

Para optimizar rendimiento, mejor usarlo en elementos cuya visibilidad va a modificarse a menudo

  <!-- HTML -->
  <div id="app">
    <h1 v-show="ok">Hello!</h1>
  </div>
  
  <!-- JS -->
  <script>
  const app = new Vue({
    el: '#app',
    data: {
      ok: true
    }
  })
  </script>

Directivas: v-if


v-if oculta o muestra el elemento DOM al que se le asigna en función del valor booleano del modelo que seleccionemos

A diferencia de v-show, cuando la propiedad del modelo a la que se le vincule sea false, el elemento se elimina del DOM

Para optimizar rendimiento, mejor usarlo en elementos cuya visibilidad NO va a modificarse a menudo (una única evaluación)

Si se quiere ocultar/mostrar un bloque HTML, usar la etiqueta template

  <!-- HTML -->
  <div id="app">
    <h1 v-if="ok">Hello!</h1>
  </div>
  
  <!-- JS -->
  <script>
  const app = new Vue({
    el: '#app',
    data: {
      ok: true
    }
  })
  </script>

Directivas: v-else


v-else ofrece mostrar un elemento DOM alternativo si nuestro v-if es false

v-else tiene que ir SIEMPRE a continuación del v-if

<!-- HTML -->
<div id="app">
  <h1 v-if="ok">Ok es igual a true</h1>
  <h1 v-else>Ok es igual a false</h1>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    ok: false
  }
})
</script>

Directivas: v-else-if


v-else-if permite mostrar un elemento o grupo del DOM entre varios en función de la evaluación de una condición del modelo

v-if, v-else-if y v-else tiene que ir SIEMPRE declaradas de manera consecutiva

<!-- HTML -->
<div id="app">
  <h1 v-if="edad < 18">
    Usuario menor de edad
  </h1>
  <h1 v-else-if="edad >= 18 && edad < 65">
    Usuario en edad laboral
  </h1>
  <h1 v-else-if="edad >= 65">
    Usuario jubilado
  </h1>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    edad: 17
  }
})
</script>

Directivas: v-for


v-for permite iterar sobre un rango numérico, un array, un objeto o un array de objetos

Requiere la sintaxis especial 'item in items', donde items es la fuente de datos e item es un alias de cada elemento iterado.

<!-- HTML -->
<div id="app">
  <ol>
    <li v-for="tarea in tareas">
      {{ tarea.text }}
    </li>
  </ol>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    tareas: [
      { text: 'Aprender Vue' },
      { text: 'Hacer mi primera aplicación Vue' }
    ]
  }
})
</script>

Directivas: v-for sobre un rango numérico


v-for acepta un número para iterar. El elemento del DOM se repetirá tantas veces como se indique en el número usado.

<!-- HTML -->
<div id="app">
  <ul>
    <li v-for="num in 10">
      {{ num }} por 10 igual a {{ num * 10 }}
    </li>
  </ul>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app'
})
</script>

v-for en este caso itera sobre los elementos de un array definido en el data de la instancia Vue.

Gracias a la reactividad, si el array varía, el v-for reacciona desplegando esa variación en el DOM

<!-- HTML -->

Directivas: v-for sobre un objeto


v-for en este caso itera sobre los elementos de un objeto definido en el data de la instancia Vue.

Gracias a la reactividad, si el objeto varía, el v-for reacciona desplegando esa variación en el DOM

Dado que el objeto se representa por un par de clave-valor, los usaremos como argumento en la declaración para representarlos

Como Vue mapea las propiedades del objeto, para añadir reactividad y data-binding a una nueva propiedad del objeto, tendremos que utilizar la propiedad set Vue.set(app.phone, 'condition', 'new');

<!-- HTML -->
<div id="app">
  <ol>
    <li v-for="(value, key) in phone">
      El valor de la propiedad {{ key }} es {{ value }}
    </li>
  </ol>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    phone: {
      name: 'LG',
      model: 'G6',
      color: 'Negro'
    }
  }
})
</script>

Directivas: v-for sobre un array de objetos


v-for en este caso itera sobre los objetos de un array definido en el data de la instancia Vue.

Gracias a la reactividad, si el array varía, el v-for reacciona desplegando esa variación en el DOM

Vue envuelve los métodos de mutación de un array observado para lanzar los cambios de la vista. Estos métodos son: push(), pop(), shift(), unshift(), splice(), sort(), reverse()

<!-- HTML -->
<div id="app">
  <ol>
    <li v-for="(phone, index) in phones" :key="phone.id">
      {{ index }} Este teléfono es un {{ phone.name }} {{ phone.model }} de color {{ phone.color }}
    </li>
  </ol>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    phones: [
      { id: 001, name: 'LG', model: 'G6', color: 'Negro' },
      { id: 002, name: 'iPhone', model: '8', color: 'Plata' },
      { id: 003, name: 'Samsung', model: 'S8', color: 'Dorado' }
    ]
  }
})

setTimeout(function() {
	app.phones.push({id: 004, name: 'Huawei', model: 'P8', color: 'Blanco'})
}, 1000)
</script>

Directivas: v-on


Escucha varios eventos DOM (click, keyup, submit)

Mediante los modificadores de evento (.stop, .prevent, .capture, .self y .once) se puede cambiar el comportamiento del mismo.

Para el evento de tecla pulsada keyup existe un mapeo configurable de teclas y keycodes

<!-- HTML -->
<div id="app">
  <button v-on:click="addOne">Pulsado {{count}} veces.</button>
  <button @click.once="resetCount">Resetea Count una vez por el uso del modificador .once</button>
  <button @click="resetCount">Resetea Count</button>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    count: 0
  },
  methods: {
    addOne: function () {
      this.count++;
    },
    resetCount: function () {
      this.count = 0;
    }
  }
})
</script>

const app = new Vue({

      color: 'red',

Directivas: v-html


Dado que la interpolación de doble llave interpreta el valor de data como texto plano, para bindear texto html se requiere esta directiva.

El contenido del elemento se reemplaza por el valor de la propiedad, interpretado como HTML, por lo que los data-bindings se pierden.

No asociar contenido HTML de fuentes no controladas para evitar inyecciones XSS en vuestro código.

<!-- HTML -->
<div id="app">
  <div v-html="rawHtml"></div>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    rawHtml: '<a href="https://vignette.wikia.nocookie.net/filthy-frank/images/8/8d/516c32f08e03d.png">Foto de Nicholas Cage</a>'
  }
})
</script>

Estilos


Podemos bindear clases CSS y estilos en linea con propiedades del modelo

Las clases gestionadas dinámicamente mediante la directiva v-bind:class se añaden a las clases CSS definidas por el usuario en el atributo class.

Podeis agrupar clases o estilos en línea en objetos dentro de data si os resulta mas cómodo.

<!-- CSS -->
p { color: red; }

.bold { font-weight: bold; }

<!-- HTML -->
<div id="app">
  <p class="bold" :class="{active: isActive}">Estado del cliente</p>
  <button @click="toggleIsActive">Cambiar isActive</button>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    isActive: true
  },
  methods: {
    toggleIsActive() { this.isActive = !this.isActive }
  }
})
</script>

Transiciones Enter/Leave (I)


Vue provee una variedad de formas para aplicar efectos de transición cuando los elementos se insertan, actualizan o eliminan del DOM.

Mediante el componente <transition> permite añadir transiciones de entrada y salida para cualquier elemento en los siguientes casos:

  1. Renderizado condicional (usando v-if)
  2. Display condicional (usando v-show)
  3. Componentes dinámicos
  4. Nodos de componente raíz
<!-- CSS -->
.fade-enter-active, .fade-leave-active { transition: opacity .5s }
.fade-enter, .fade-leave-to { opacity: 0 }

<!-- HTML -->
<div id="app">
  <button v-on:click="show = !show">Toggle</button>
  <transition name="fade">
    <p v-if="show">Fading</p>
  </transition>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    show: true
  }
})
</script>

Propiedades computadas (Computed properties)


Las utilizaremos cuando necesitemos una expresión evaluada pero su lógica sea demasiado compleja como para declararla en el template.

Su escenario ideal son transformaciones directas de datos.

No modifican el valor del dato desde el que se realiza la transformación.

Aunque, como vemos en el ejemplo, se puede obtener el mismo resultado con un método, los resultados de los métodos no se cachean y se vuelven a ejecutar siempre aunque la propiedad no cambie.

<!-- HTML -->
<div id="app">
  <p>Original message: "{{ message }}"</p>
  <p>Computed reversed message: "{{ reversedMessage }}"</p>
  <p>Reversed message: "{{ reverseMessage() }}"</p>
</div>

<!-- JS -->
<script>
const app = new Vue({
  el: '#app',
  data: {
    message: 'Hola'
  },
  computed: {
    reversedMessage: function () { //reversedMessage() { ES6 válido también
      return this.message.split('').reverse().join('');
    } 
  },
  methods: {
    reverseMessage: function () {
      return this.message.split('').reverse().join('')
    }
  }
})
</script>

Watches


Modo mas genérico de observar y reaccionar a cambios en el data en una instancia vue.

El contexto mas eficaz de aplicación son transformaciones mas complejas dependientes de terceros (llamadas asíncronas a APIs).

Ver ejemplo de Watch de la documentación de Vue

<!-- HTML -->
<div id="app">
  <div id="demo">{{ fullName }}</div>
  <input v-model="firstName" placeholder="Nombre" />
  <input v-model="lastName" placeholder="Apellido" />
</div>

<!-- JS -->
<script>
  var vm = new Vue({
    el: '#demo',
    data: {
      firstName: 'Foo',
      lastName: 'Bar',
      fullName: 'Foo Bar'
    },
    watch: {
      firstName: function (val) {
        this.fullName = val + ' ' + this.lastName
      },
      lastName: function (val) {
        this.fullName = this.firstName + ' ' + val
      }
    }
  })
</script>

Watches vs Computed properties


La mayoría de las ocasiones, usar Computed es mas eficiente que Watch

<!-- HTML -->
<div id="app">
  <div id="demo">{{ fullName }}</div>
  <input v-model="firstName" placeholder="Nombre" />
  <input v-model="lastName" placeholder="Apellido" />
</div>

<!-- JS -->
<script>
  var vm = new Vue({
    el: '#demo',
    data: {
      firstName: 'Foo',
      lastName: 'Bar'
    },
    computed: {
      fullName: function () {
        return this.firstName + ' ' + this.lastName
      }
    }
  })
</script>

Filtros


Se usan para aplicar formato específico a salida de texto.

Se sitúan al final de la expresion interpolada en el template antecedidos por el simbolo pipe |, pudiendo encadenarse.

{{ message | filter1 | filterN }}.

<!-- HTML -->
<div id="app">
  <div id="demo">{{ message | reverse | uppercase }}</div>
  <input v-model="message" />
</div>

<!-- JS -->
<script>
  var vm = new Vue({
    el: '#demo',
    data: {
      message: 'talavera'
    },
    filters: {
      uppercase: function (value) {
        if (!value) return ''
        value = value.toString()
        return value.toUpperCase()
      },
      reverse: function (value) {
        if (!value) return ''
        return value.split('').reverse().join('')
      }
    }
  })
</script>

Componentes


Ayudan a extender elementos HTML básicos con el fín de encapsular código reutilizable.

Son elementos personalizados (custom elements) a los que el compilador de Vue adjunta un comportamiento.

En algunos casos, y al igual que en Polymer 1.x, pueden aparecer como elementos HTML nativos extendidos con el atributo especial 'is'.

<!-- HTML -->
<div id="app">
  <mi-componente></mi-componente>
</div>

<!-- JS -->
<script>
  Vue.component('mi-componente', {
    template: '<div>El contenido de mi componente!</div>'
  })
  var vm = new Vue({
    el: '#app'
  })
</script>

Composición con componentes


Los componentes están pensados para ser usados conjuntamente, estableciendo jerarquías y relaciones entre ellos. Por ello, un componente padre A puede instanciar dentro de el un componente hijo B.

Hay que evitar acoplar el funcionamiento de los componentes para asegurar su aislamiento e interoperabilidad.

La comunicación entre componentes sigue una regla clara, "Props down, events up" (Propiedades hacia abajo, eventos hacia arriba).

Composición con componentes (II)


El sistema de componentes es una abstracción que nos permite construir aplicaciones de gran tamaño como una suma de pequeñas piezas, como un lego.

Cualquier interface de aplicación puede abstraerse en un árbol jerárquico de componentes.

En Vue, un componente es esencialmente una instancia Vue con una serie de opciones predefinidas.

De esta manera, podemos estructurar una app en componentes de la siguiente manera:

<!-- HTML -->
<div id="app">
  <app-nav></app-nav>
  <app-content>
    <one-component></one-component>
    <another-component></another-component>
  </app-content>
  <app-sidebar>
    <component-x></component-x>
    <component-y></component-y>
    <component-x></component-x>
  </app-sidebar>
</div>

Custom events


Un componente hijo se comunica con su padre a través de un custom event.

La interface de eventos permite escuchar un evento usando $on(eventName) -addEventListener nativo- y lanzar un evento usando $emit(eventName) -dispatchEvent nativo-

Un componente padre puede escuchar un evento lanzado desde un hijo usando v-on directamente en el template donde se instancia el componente hijo.

<!-- HTML -->
<div id="counter-event-example">
  <p>{{ total }}</p>
  <button-counter v-on:increment="incrementTotal"></button-counter>
  <button-counter v-on:increment="incrementTotal"></button-counter>
</div>

<!-- JS -->
<script>
  Vue.component('button-counter', {
    template: '<button v-on:click="incrementCounter">{{ counter }}</button>',
    data: function () { 
      return { counter: 0 } 
    },
    methods: {
      incrementCounter: function () { 
        this.counter += 1
        this.$emit('increment')
      }
    },
  })
  new Vue({
    el: '#counter-event-example',
    data: { 
      total: 0 
    },
    methods: { 
      incrementTotal: function () { this.total += 1 } }
  })
</script>

Event bus


Un bus de eventos permitirá comunicar componentes sin vinculación jerárquica.

En escenarios simples, definiremos una instancia vue vacía como bus central de eventos

En aplicaciones de gran tamaño, el bus de eventos puede no ser suficiente y precisaremos un patrón de manejo de estado, como el que nos facilita Vuex.


<!-- JS -->
<script>
  var bus = new Vue()
  // in component A's method
  bus.$emit('id-selected', 1)
  // in component B's created hook
  bus.$on('id-selected', function (id) {
    // ...
  })
</script>

Mixins


Los mixins son una manera flexible de distribuir funcionalidad reutilizable entre componentes Vue.

Cuando un componente importa un mixin, todas las opciones del mixin se mezclan con las opciones del propio componente.

Si un mixin y un componente contienen opciones que se solapan en sus respectivos hooks de ciclo de vida, se mergean. De este modo, en el caso de que mixin y componente compartan hooks, se llamarán primero los del mixin.

Si un mixin y un componente compartieran métodos con el mismo nombre, los métodos del componente tendrán prioridad sobre los del mixin.


<!-- JS -->
<script>
  // define a mixin object
  var myMixin = {
    created: function () {
      this.hello()
    },
    methods: {
      hello: function () {
        console.log('hello from mixin!')
      }
    }
  }
  // define a component that uses this mixin
  var Component = Vue.extend({
    mixins: [myMixin]
  })
  var component = new Component() // => "hello from mixin!"
</script>

Slots


Los slots permiten distribuir contenido de manera específica insertando dentro del elemento 'slot' del componente hijo el contenido declarado en la instancia del componente hijo hecha en el componente padre.

El contenido distribuido se compila en el scope del componente padre.

El contenido distribuido se descartará a menos que el componente hijo contenga al menos un slot. En ese caso, el contenido reemplazará al del slot.

Un componente puede declarar varios slots diferenciándolos mediante el uso del atributo name.

<!-- HTML -->
<!-- app-layout component´s template -->
<div class="container">
  <header>
    <slot name="header"></slot>
  </header>
  <main>
    <slot></slot>
  </main>
  <footer>
    <slot name="footer"></slot>
  </footer>
</div>

<!-- Parent´s markup -->
<app-layout>
  <h1 slot="header">Here might be a page title</h1>
  <p>A paragraph for the main content.</p>
  <p>And another one.</p>
  <p slot="footer">Here's some contact info</p>
</app-layout>

Single file components


Ya hemos visto que la manera habitual, sencilla y práctica de crear componentes en vue es usando Vue.component('nombre-del-componente', {})

De todos modos, nos fuerza a escribir el html dentro del método o a declarar múltiples templates en nuestro HTML.

Para solucionar esto, Vue provee una estructuración de componentes como archivos independientes terminados con el sufijo .vue.

Estos archivos alojarán la vista (template), la lógica (js) y el estilo (css) del componente, creando un elemento reutilizable e interoperable. 

<!-- Vue component declaration -->
Vue.component('user-info', {
  props: ['image', 'fullName'],
  template: `
    <div>
      <img width="100" v-bind:src="image" v-bind:alt="fullName"/>
      <h2>{{ fullName }}</h2>
    </div>
  `,
})
<!-- Single file component declaration -->
<template>
  <div>
    <img width="100" v-bind:src="image" v-bind:alt="fullName"/>
    <h2>{{ fullName }}</h2>
  </div>
</template>
<script>
export default {
  name: 'user-info',
  props: {
      image: String,
      fullName: String
  }
}
</script>
<style scoped>
  h2 { font-size: 18pt; }
</style>
